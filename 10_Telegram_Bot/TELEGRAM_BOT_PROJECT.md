# Telegram Bot Project

## Purpose

The Telegram Bot brings operational reports directly to the teams instead of forcing users to search folders, open shared drives or ask for the latest file.

## What it demonstrates

- Automated report delivery.
- Centralized access to vessel reports and schedules.
- Reusable Python structure.
- Potential for daily or intraday operational updates.

## Why it matters for the data migration

Once the data layer is centralized, Telegram becomes a distribution channel: the bot can notify teams when new dashboards are ready, send links to updated reports and reduce manual follow-up.

## Files included

- `bot_telegram_maritimo.py`: main maritime bot script.
- `config.py`: configuration helper.
- `vessel_operation_base.json`: vessel operation configuration sample.
- `Demo_Service_Schedule.html`: example report exposed through the bot.
- `requirements.txt`: Python dependencies.
- `run_bot.bat`: execution shortcut.

## Meeting angle

This is the bridge between data processing and operational adoption. Dashboards are useful; automated delivery makes them part of the daily routine.
