Built by operations,
for operations

Every system shown here started with the same frustration: too many files, too many manual steps, and a decision window that was closing before the numbers were even ready. I built these tools to solve that - not as side projects, but as daily operational necessity.

The stack is Python for processing, HTML for delivery and SQL for the data layer. The approach is always the same: take what the system exports, remove the manual steps in between, and deliver something that any team member can read and act on before the deadline hits.

The result is a connected intelligence layer across booking control, capacity planning, gate-in monitoring, stowage inspection and governance - all running from the same operational data.

Booking Control Board
Complete booking visibility by vessel - container mix by type (DC20, HC40, RH40, FR40), over/under status per dimension (TEUs, tons, plugs), port breakdown, customer performance, deadlines and pending cargo.
